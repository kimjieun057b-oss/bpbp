# [PRD] 펜션/캠핑장 캘린더 예약 시스템 보일러플레이트

## 1. 개요 및 목표 (Overview & Goals)
* **개요**: 캠핑장 사이트 및 펜션 객실 예약 시 가장 범용적으로 사용되는 달력(Calendar) 중심의 원페이지(One-Page) 예약 기능 보일러플레이트 제작.
* **목표**:
 * 외부 API 의존 없이 Next.js + Tailwind CSS + Supabase 스택만으로 즉시 구동되는 독립형 예약 엔진 구축.
 * 프로젝트마다 디자인 시스템이 변경되므로 스타일은 미니멀하게 유지하되, 연박 선택 로직 및 실시간 잔여 객실 계산이라는 핵심 기능에 집중.


## 2. 핵심 기능 (Core Features)
* **캘린더 연박 선택 엔지니어링 (Calendar Interaction)**
 * 단일 날짜 선택이 아닌 '체크인 날짜 클릭 ➡️ 체크아웃 날짜 클릭'으로 이어지는 기간 선택 인터랙션.
 * 마우스 호버(Hover) 시 선택 중인 기간의 중간 날짜들을 시각적으로 하이라이트 처리.
* **실시간 잔여 객실 검증 (Inventory & Overbooking Protection)** 
 * 사용자가 달력에서 기간을 선택하면, Supabase를 조회하여 해당 기간 내에 이미 예약이 확정된 객실을 실시간으로 disabled(선택 불가) 처리.
 * 동일 객실에 대한 중복 예약(Overbooking) 원천 차단.
* **유연한 옵션 및 인원 추가 (Pricing Options)**
 * 객실별 기준 인원/최대 인원 초과 시 추가 금액 계산 로직.
 * 바비큐(숯불), 불멍(장작) 등 캠핑/펜션 특화 부가 서비스 선택 상태 관리.
* **Supabase 데이터베이스 연동 (DB Integration)**
 * 객실 정보 테이블(rooms), 예약 확정 데이터 테이블(bookings)의 관계형 모델링.
 * 트랜잭션 개념을 적용하여 결제/예약 완료 단계에서 최종 정합성 검증.


## 3. 화면 구성 및 위계 (Layout)
메인 화면은 캘린더가 전체 레이아웃의 중심(Main Axis)을 잡고, 우측 또는 하단에 상태 변화에 따른 정보 창이 붙는 구조로 시맨틱하게 마크업합니다. (table 태그 배제, Flex/Grid 레이아웃 준수)

* **Left/Main Column: 메인 예약 캘린더 영역 (<section>)**
 * 상단 컨트롤러: 연도/월 이동 버튼 (이전 달, 다음 달), 현재 달 표시.
 * 요일 헤더: 일~월 고정 그리드 (grid-cols-7).
 * 달력 바디: 7열 구조의 일자 버튼 배치.
   * 스타일 상태 분기: [오늘 / 과거 날짜 / 선택된 체크인 / 선택된 체크아웃 / 기간 내부(In-between) / 예약 마감된 날짜].
* **Right/Sub Column: 예약 정보 및 옵션 검수 영역 (<aside> 또는 하단 배치)**
 * 기간 표시기: 미선택 시 "날짜를 선택해 주세요" ➡️ 선택 시 "0월 0일(수) ~ 0월 0일(금) / 2박" 실시간 변환.
 * 객실/사이트 선택: 사용 가능한 객실 목록 렌더링 (라디오 버튼 또는 커스텀 카드 형태).
 * 옵션 및 인원 설정: 인원 추가(+, - 버튼), 부가 서비스 체크박스.
 * 최종 금액 수수료 요약 및 예약하기 버튼: 포인트 컬러를 활용한 고정 버튼.


## 4. 사용자 시나리오 (User Scenario)
 * [단계 1: 탐색] 유저가 예약 페이지 진입 (달력에 이번 달 현황 로드)
 * [단계 2: 체크인 선택] 원하는 입실 일자(Date) 클릭 (해당 날짜에 '체크인' 뱃지 표시)
 * [단계 3: 기간 지정] 마우스를 움직여 퇴실 일자 위로 올리면 기간이 하이라이트되며, 클릭 시 '체크아웃' 확정
 * [단계 4: 객실 및 옵션 매칭] 선택한 기간 동안 '예약 가능한 방' 목록이 노출되며, 인원 및 바비큐 옵션 체크
 * [단계 5: 최종 검증] '예약하기' 클릭 시 Supabase에서 그 찰나에 다른 사람이 먼저 예약했는지 최종 validation
 * [단계 6: 완료] 예약 완료 및 마이페이지/확인 뷰로 라우팅


## 5. DB 아키텍처 가이드 (Supabase Table Schema)
보일러플레이트가 정상 작동하기 위해 Supabase에 미리 심어두어야 할 최소한의 핵심 테이블 뼈대 스펙입니다.

1) rooms (객실/사이트 정보)
컬럼명 | 타입 | 설명
id | uuid(PK) | 고유 식별자
name | text | 방 이름 (예: A-1 데크, 파인룸)
base_price | integer | 평일 기본 요금
max_people | integer | 최대 수용 인원

2) bookings (예약 확정 데이터)
컬럼명 | 타입 | 설명   
id | uuid(PK) | 고유 식별자
room_id | uuid(FK) | rooms.id 참조
check_in | date | 체크인 날짜(YYYY-MM-DD)
check_out | date | 체크아웃 날짜(YYYY-MM-DD)
status | text | 예약 상태 (confirmed, pending, cancelled)
